# Capsa Consulting Examples

## Example 1: ERP Delay

Input:

> The ERP project is delayed. The client says the software is too rigid. The vendor says the client keeps changing requirements.

Response:

```text
Client signal:
The ERP project is exposing unresolved process decisions.

What is really happening:
The client has not agreed on the operating process the software is supposed to support.

Operating gap:
Requirements keep changing because decision rights and process ownership are unclear.

Decision needed:
Decide who owns each core process and who has authority to approve changes.

Recommended next moves:
1. List the top 5 processes blocking the build.
2. Assign one business owner to each process.
3. Hold a decision session for each process.
4. Freeze approved decisions for the next build cycle.
5. Keep a visible decision log.

Owner:
Executive sponsor owns decision speed. Process owners own process clarity.

Rhythm:
Twice-weekly 45-minute decision meeting until the blockers are cleared.

Client artifact:
Process decision log with owner, decision, date, impact, and open questions.

How to review:
At the end of 2 weeks, measure reduced open decisions and fewer requirement reversals.
```

## Example 2: Dashboard Adoption

Input:

> The client paid for a dashboard, but managers still ask analysts for manual reports.

Response:

```text
Client signal:
Managers do not trust the dashboard enough to use it for decisions.

What is really happening:
The dashboard may be technically complete, but it is not embedded in the management rhythm.

Operating gap:
No one has defined which decisions the dashboard supports or when leaders should use it.

Decision needed:
Choose the operating meeting where the dashboard becomes the source of truth.

Recommended next moves:
1. Identify the 3 decisions the dashboard should support.
2. Remove metrics that do not support those decisions.
3. Assign metric owners.
4. Use the dashboard in the next 4 operating reviews.

Owner:
Business sponsor owns adoption. Analytics owns data support.

Rhythm:
Weekly operating review using the dashboard as the agenda.

Client artifact:
Dashboard decision map: metric, decision, owner, review cadence.

How to review:
After 4 reviews, check whether manual report requests decreased.
```
